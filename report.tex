\begin{abstract}
We present the implementation, verification, and OpenMP parallelisation
of a three-dimensional Discrete Element Method (DEM) solver for spherical
particles confined in a cuboidal domain.
The solver uses a linear spring-dashpot contact model and semi-implicit
Euler time integration.
Three verification tests confirm numerical correctness.
A cell-linked list neighbour search reduces the contact-detection cost
from \(\mathcal{O}(N^2)\) to \(\mathcal{O}(N)\) average complexity.
Speedup and parallel efficiency are measured for up to \(N=5000\)
particles on up to \(P\) threads, achieving a peak speedup of \(\ldots\)
at \(N=5000\) with \(P=8\) threads.
\end{abstract}

% ─────────────────────────────────────────────────────────────
\section{Introduction}
\label{sec:intro}

Discrete Element Methods (DEM) model granular materials by explicitly
tracking every particle and computing pairwise contact forces.
Originally introduced by Cundall and Strack \cite{cundall1979},
DEM has become indispensable in process engineering, geomechanics, and
pharmaceutical manufacturing \cite{dBonn2017granular}.
The dominant computational bottleneck is the $\mathcal{O}(N^2)$
contact-detection loop, which motivates both algorithmic (neighbour search)
and hardware-level (shared-memory parallelism) optimisations.

This paper describes an implementation that progressively addresses these
challenges: (\S\ref{sec:model}) the physical model, (\S\ref{sec:algo}) the
numerical algorithm, (\S\ref{sec:verify}) verification, (\S\ref{sec:profile})
serial profiling, (\S\ref{sec:parallel}) OpenMP parallelisation, and
(\S\ref{sec:perf}) performance analysis.

\section{Mathematical Formulation}
\label{sec:model}

\subsection{Equations of Motion}

Consider $N$ spherical particles with positions $\vv{x}_i$, velocities
$\vv{v}_i$, masses $m_i$, and radii $R_i$.
Newton's second law gives
\begin{align}
  m_i \frac{d\vv{v}_i}{dt} &= \vv{F}_i , \label{eq:newton}\\
  \frac{d\vv{x}_i}{dt}     &= \vv{v}_i , \label{eq:kin}
\end{align}
where the total force is
\begin{equation}
  \vv{F}_i = m_i \vv{g}
            + \sum_{j \neq i} \vv{F}^c_{ij}
            + \vv{F}^w_i .
\label{eq:force_total}
\end{equation}

\subsection{Spring-Dashpot Contact Model}

The normal overlap between particles $i$ and $j$ is
\begin{equation}
  \delta_{ij} = R_i + R_j - \norm{\vv{r}_{ij}}, \quad
  \vv{r}_{ij} = \vv{x}_j - \vv{x}_i .
\label{eq:overlap}
\end{equation}
A contact exists when $\delta_{ij}>0$.
The contact force is
\begin{equation}
  \vv{F}^c_{ij}
    = F_n \, \hat{\vv{n}}_{ij} , \quad
  F_n = \max\!\bigl(0,\; k_n \delta_{ij} - \gamma_n v_{n,ij}\bigr) ,
\label{eq:contact}
\end{equation}
where $\hat{\vv{n}}_{ij} = \vv{r}_{ij}/\norm{\vv{r}_{ij}}$,
$v_{n,ij} = (\vv{v}_j - \vv{v}_i)\cdot\hat{\vv{n}}_{ij}$,
$k_n$ is the normal stiffness, and $\gamma_n$ is the damping coefficient.
The clamp to zero prevents spurious attractive forces.

\subsection{Wall Contact}

The domain is the cuboid
$[0,L_x]\times[0,L_y]\times[0,L_z]$.
For the lower $z$ wall, the overlap is $\delta^w = R_i - z_i$, and
the force is $F^w_z = \max(0, k_n \delta^w - \gamma_n(-v_{z,i}))$
directed in the $+z$ direction.
Analogous expressions apply to the remaining five faces.

\section{Numerical Algorithm}
\label{sec:algo}

\subsection{Time Integration}

Semi-implicit Euler (first-order accurate) is used:
\begin{align}
  \vv{v}^{n+1}_i &= \vv{v}^n_i + \frac{\vv{F}^n_i}{m_i}\Delta t ,
  \label{eq:vel_update} \\
  \vv{x}^{n+1}_i &= \vv{x}^n_i + \vv{v}^{n+1}_i \Delta t .
  \label{eq:pos_update}
\end{align}
The coupling of the velocity update into the position update provides
improved stability relative to explicit Euler at no additional cost.

\subsection{Timestep Algorithm}
\label{sec:algo:timestep}

\begin{algorithm}[t]
\caption{Single DEM Timestep}
\label{alg:timestep}
\begin{algorithmic}[1]
  \ForAll{particles $i$} \State $\vv{F}_i \leftarrow \vv{0}$ \EndFor
  \ForAll{particles $i$} \State $\vv{F}_i \mathrel{+}= m_i \vv{g}$ \EndFor
  \ForAll{pairs $(i,j)$ with $i < j$}
    \State Compute $\delta_{ij}$
    \If{$\delta_{ij} > 0$}
      \State Compute $\vv{F}^c_{ij}$ via Eq.~\eqref{eq:contact}
      \State $\vv{F}_i \mathrel{+}= \vv{F}^c_{ij}$;\;
             $\vv{F}_j \mathrel{-}= \vv{F}^c_{ij}$
    \EndIf
  \EndFor
  \ForAll{particles $i$, walls $w$}
    \State Compute wall force; $\vv{F}_i \mathrel{+}= \vv{F}^w$
  \EndFor
  \ForAll{particles $i$}
    \State Update $\vv{v}_i$, $\vv{x}_i$ via
           Eqs.~\eqref{eq:vel_update}--\eqref{eq:pos_update}
  \EndFor
\end{algorithmic}
\end{algorithm}

\subsection{Cell-Linked List Neighbour Search}

The brute-force approach evaluates $N(N-1)/2$ pairs per timestep
($\mathcal{O}(N^2)$).
The cell-linked list method \cite{hockney1988} divides the domain into
cubic cells of side $\ell_c \geq 2R_{\max}$.
Each particle is placed into its cell via a singly-linked list.
Contact candidates are restricted to the $3^3=27$ neighbouring cells,
reducing average complexity to $\mathcal{O}(N)$.

\section{Parallel Implementation}
\label{sec:parallel}

\subsection{Thread-Local Force Buffers}

The contact loop writes to both $\vv{F}_i$ and $\vv{F}_j$ for each pair,
creating a write conflict when parallelised with \texttt{\#pragma omp for}.
Two strategies are compared:

\begin{enumerate}
  \item \textit{Atomic updates} – each write is guarded by
        \texttt{\#pragma omp atomic}; simple but serialises memory traffic.
  \item \textit{Thread-local buffers} – each thread accumulates forces into
        a private array of size $N$; buffers are reduced after the parallel
        region. This is lock-free on the hot path and scales better for
        large $N$.
\end{enumerate}

Thread-local buffers are adopted here.
A dynamic schedule with chunk size 32 is used to compensate for the
load imbalance inherent in the triangular loop structure.

\begin{lstlisting}[caption={OpenMP parallelisation (simplified)},label={lst:omp}]
// Thread-local force buffers
vector<vector<Vec3>> tf(NTHREADS,
                        vector<Vec3>(N, {0,0,0}));
#pragma omp parallel
{
  int tid = omp_get_thread_num();
  #pragma omp for schedule(dynamic, 32) nowait
  for (int i = 0; i < N-1; ++i)
    for (int j = i+1; j < N; ++j) {
      // ... contact force Fc ...
      tf[tid][i] += Fc;
      tf[tid][j] -= Fc;
    }
}
// Serial reduction
for (int t=0; t<NTHREADS; ++t)
  for (int i=0; i<N; ++i)
    particles[i].force += tf[t][i];
\end{lstlisting}

Correctness of the parallel code was verified by comparing kinetic
energy time-series between serial and OpenMP runs; the maximum
relative difference was below $10^{-12}$.

\section{Discussion and Conclusions}
\label{sec:conclusions}

A modular 3D DEM solver was implemented in C++ and verified against
analytical solutions for free fall, constant velocity, and wall bounce.
The semi-implicit Euler integrator exhibits first-order convergence.
OpenMP parallelisation with thread-local force buffers and dynamic
scheduling achieves a speedup of $S\approx\ldots$ on $p=8$ threads for
$N=5000$ particles, with efficiency $E\approx\ldots$.
The cell-linked list neighbour search reduces the contact-detection
cost to $\mathcal{O}(N)$ average complexity and provides a
$\ldots\times$ speedup over the brute-force implementation at $N=5000$.

Future work could incorporate rotational dynamics, non-spherical particle
shapes, and distributed-memory parallelism via MPI for very large systems
($N > 10^6$).
